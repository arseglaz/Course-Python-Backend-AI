1. Почему кортеж может быть ключом словаря, а список обычно нет?
2. Когда кортеж лучше списка?
3. Что такое распаковка кортежа?
4. Почему методов у кортежа меньше, чем у списка?

```
1) Ключь словаря должен быть `хэшируемым`, то есть иметь фиксированный хэш, который не может поменяться со временем.
`Хэш` - уникальный цифровой "идентификатор"
А список можно изменить, значит хэш стал бы невалидным. `Python` вообще не дает хэшировать списки. 

2) Когда данные не должны меняться ( координаты, RPG цвет, неизменяемые данные пользователя ), Использование ключей словаря

3) Распаковка кортежа - это присвоение нескольким элементам кортежа нескольким переменым . Главное, чтобы количество элементов совпадало с количеством переменымх

```Python
name, age, city = ("Arseniy", 21, "Israel")
```

4) Потому что кортеж неизменяемый, все методы кортежа только помогают вывести данные из кортежа, потому что ничего другого сделать нельзя, в отличии от списков 

Методы кортежа:
```Python
.count(x) - сколько раз встречается обьект x
.index(x) - на каком индексе x
